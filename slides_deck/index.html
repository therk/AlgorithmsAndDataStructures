Linked List

  • Linked lists are a dynamic data structure, which can grow and be pruned, allocating and deallocating memory while the program is running.
  • Insertion and deletion node operations are easily implemented in a linked list.
  • Dynamic data structures such as stacks and queues can be implemented using a linked list.
  • There is no need to define an initial size for a linked list.
  • Items can be added or removed from the middle of list.
  • They use more memory than arrays because of the storage used by their pointers.
  • Nodes in a linked list must be read in order from the beginning as linked lists are inherently sequential access.

public class MyLinkedList<Item> {

  private Node<Item> node = new Node<Item>();

  boolean hasNext() {
    return node.next == null;
  }

  void setItem(Item item) {
    node.item = item;
  }

  void addNext(Item item) {
    Node<Item> oldNode = node;
    node = new Node<>();
    node.item = item;
    node.next = oldNode;
  }

  Node getNext() {
    return node.next;
  }
}

Generic Stack: Linked-list implementation

public class MyStack<Item>{ 
  private Node<Item> first = null;

  public boolean isEmpty() {
    return first == null;
  }

  public void push(Item item) {
    Node<Item> oldFirst = first;
    first = new Node<>();
    first.item = item;
    first.next = oldFirst;
  }

  public Item pop() {
    Item item = first.item;
    first = first.next;
    return item;
  }
}

Generic Stack: Fixed Array implementation

public class FixedCapacityStack<Item>{ 
    private Item[] items; 
    private int currentLocation = 0; 
    public FixedCapacityStack(int capacity) { 
        items = (Item[]) new Object[capacity]; 
    } 
    public boolean isEmpty() { 
        return currentLocation == 0; 
    } 
    public void push(Item item) { 
        items[currentLocation++] = item; 
    } 
    public Item pop() { 
        return items[--currentLocation]; 
    }
}

Fixing challenges with Array based Stack

  • Overflow and underflow.
    • Underflow: throw an exception if pop from an empty stack.
    • Overflow: use resizing array for array implementation.
  • Null items: We allow null items to be inserted.
  • Loitering: Holding a reference to an object when it is no longer needed.

Resizing Array Stack Implementation

public ResizingArrayStack() {

}

public void push(Item item) {

    s[currentLocation++] = item;
}

private void resize(int capacity) {
    Object[] copy = new Object[capacity];
    for (int i = 0; i < currentLocation; i++)
        copy[i] = items[i];
    items = copy;
}

If the array is full, create a new array of twice the size, and copy items.

Dijkstra's two-stack algorithm

  • Value: push onto the value (operand) stack.
  • Operator: push onto the operator stack.
  • Left parenthesis: ignore
  • Right parenthesis: pop operator and two values; push the results of applying that operator to those values onto the value stack.
